What are they?

Stack

Last In, First Out

A stack is like a pile of plates — you can only add or remove from the top. The last item added is always the first one taken out.

push(10) → [10] push(20) → [10, 20] pop() → 20 ← top first
Queue

First In, First Out

A queue is like a line at a counter — the first person to join is the first to be served. Order is always preserved.

enqueue(10) → [10] enqueue(20) → [10, 20] dequeue() → 10 ← front first

Time Complexity

OperationStackQueueWhy
Push / EnqueueO(1)O(1)Always added at one end — no shifting needed
Pop (top)O(1)—Direct access to the top element
Dequeue (front)—O(n)*Removing from front shifts all remaining elements
